# Character System P2 Tasks

## 概要

P2では、キャラクターシステムの運用品質と表示安定性を高める。主に名前重複制御、GUIの使い勝手、ネームタグprefixの再適用条件を整理する。

## タスク

- キャラクター名の重複制御を実装する。
  - 同一プレイヤー内で同名キャラクターを禁止するか、suffixを自動付与するかを決める。
  - SHC / THC のprefix付与後の名前でも重複しないようにする。
- GUIの拡張を行う。
  - キャラクター数がスロット上限を超える場合のページング。
  - 名前入力方式の改善（チャット入力、Anvil GUIなどの候補を比較）。
  - 作成・選択中の「処理中」表示や連打防止。
- ネームタグprefixの再適用設計を固める。
  - プレイヤーjoin、キャラクター選択、死亡、管理者revive、scoreboard差し替え時の再適用タイミングを明文化する。
  - 他プラグインがscoreboardを差し替えた場合でもSHC/THC prefixが復旧されるようにする。
- 運用向けコマンドを整理する。
  - 管理者によるキャラクター一覧確認。
  - dead/archived状態の表示・検索。
  - 必要なら強制アクティブ解除コマンドを追加する。

## 完了条件

- 同一プレイヤー内の名前重複方針が実装され、GUI/コマンド双方で同じ挙動になる。
- キャラクター数が増えてもGUIから選択・作成できる。
- SHC / THC prefixがjoin後やscoreboard変更後も安定して表示される。
- `./gradlew build` が成功する。
