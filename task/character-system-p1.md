# Character System P1 Tasks

## 概要

P1では、キャラクターシステムを「選択メタデータ」から実プレイヤー進行データへ接続する。レベル・経験値・スキルツリーなど、キャラクター別で持つべきデータを既存の `PlayerDataProvider` / `PlayerDataRepository` 方針に合わせて分離する。

## タスク

- キャラクター別に保存する進行データを棚卸しする。
  - レベル、経験値、ステータス、スキルツリー解放状況。
  - インベントリや装備、位置情報を含めるかは仕様確認後に決定する。
- `player_uuid + character_id` をキーにしたキャラクター進行データの保存層を設計する。
  - 既存 `PlayerDataProvider<T>` と整合する形で実装する。
  - STANDARD / SOFT_HARDCORE はレベルをキャラ別にし、所持金・トレーダー信用度のみ共有する。
  - TRUE_HARDCORE は進行データ、所持金、トレーダー信用度をキャラ個別にする。
- キャラクター選択時に、対象キャラクターの進行データをロードしてプレイヤーへ反映する。
- キャラクター切り替え前に、現在のアクティブキャラクターの進行データを保存する。
- キャラクター作成 + アクティブ選択をトランザクション化する。
  - 作成成功・選択失敗で孤立キャラクターや不整合が残らないようにする。
  - DB実装では同一接続・同一トランザクションで保存する。

## 完了条件

- STANDARD / SOFT_HARDCORE / TRUE_HARDCORE のレベルがキャラクター別に保存・復元される。
- キャラクター切り替えで、別キャラのレベルやスキル状態が混ざらない。
- 作成と選択の途中失敗時にDB不整合が残らない。
- `./gradlew build` が成功する。
